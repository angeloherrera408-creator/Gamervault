ARQUITECTURA & LOGICA DE NEGOCIO

Diagramación técnica de los modelos de datos, capas lógicas y middleware aplicados en la plataforma corporativa.

Middleware de Descuento Dinámico

La aplicación utiliza un patrón de diseño interceptor para la manipulación de precios en tiempo real. Cuando la tienda ejecuta el renderizado de la UI, el objeto SecurityMiddleware intercepta el costo base de la base de datos relacional y calcula un 10% de descuento automático de forma síncrona si y solo si las propiedades del cliente global registran un nivel >= 5.

Estructura de Capas del Servidor (Backend Architecture)

El ecosistema de Spring Boot se segmenta bajo el patrón arquitectónico desacoplado, asegurando el flujo unidireccional de la información:

Controller Layer Expone los Endpoints RESTful públicos y gestiona las solicitudes de recursos HTTP mediante anotaciones @RestController.
Repository Layer (JPA) Administra la abstracción de consultas relacionales abstrayendo SQL nativo a través de interfaces extendidas de JpaRepository.
Persistence Layer (H2) Motor transaccional en memoria relacional que resguarda las entidades físicas de forma volátil durante el tiempo de ejecución.

Diccionario de Datos (Modelo de Persistencia)

Campo Tipo de Dato Descripción
id Long (PK) Identificador único e incremental inyectado de forma automática en la base de datos.
